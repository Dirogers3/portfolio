W10 Notes

MDN: Validating Forms

  • Some of the built in form validations are required, minlength & maxlength, min & max(numbers), and patterns.

MDN: Using Fetch

  • Remember to treat all api calls as a promise. I always forget this and it makes my code super confusing
  • You can use fetch to upload data, by using a method key value pair.
  • Always use a catch to get the errors, just incase a api fetch failed.

Project

We have a small project that uses form validations and API fetch call.

  • The form has validations on the client side that make sure that there is something in the input box before submit.
  • After the form is submitted, there is a validation on the server side that check to see if the pokemon exists, if it doesn't it will return a red error.
  • When a correct pokemon is put in, the submit button will call an api functiont that retrieves a specific pokemon.
  • After the pokemon information is retrieved, it is then displayed in a nice format so users can read it.
  • Some examples of pokemon are bulbasaur, pikachu, charmander, or mew
